Project buLive

Live Social Video Broadcasting On The Go

Project logistics

  • Mentor: Ata Turk. Send an email to ataturk-at-bu-dot-edu if you are interested in the project.
  • Min-max team size: 3-5
  • Expected project hours per week (per team member): 8-10
  • Will the project be open source? No, we may be using some proprietary codes for building the project and some of the stuff we build may have commercial value.

Preferred past experience

Some experience in one or more of the following is preferred in possible team members:

  • Mobile app development in iOS or Android
  • Content Delivery Networks (e.g. Amazon Cloud Front)
  • Real Time Streaming (e.g. RTSP)
  • Online File Storage (e.g. Amazon S3 )

What is buLive about?

  • Ability to live broadcast from a mobile device to other mobile devices
  • User starts a video broadcast
  • It can be a public broadcast
    • Anyone can watch via “What’s streaming now!”
  • It can be a private broadcast to friends
    • (Selected) friends are notified of broadcast
  • Broadcasted videos can be stored
  • Users can also watch stored (non-live) videos
  • Videos or broadcasts can be accessed/filtered/restricted by:
    • Tags: interests of viewers
    • Locations: current location or any given location
